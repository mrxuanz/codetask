# planner-full

## Role
Drive CodeTask from a confirmed draft into Planner and plan confirmation via Test MCP only.

## Goal
1. Ensure a create_task thread has a draft that can be confirm-final'd (or reuse prior draft flow).
2. After `confirm_draft_final`, **stop conversation driving** — do not call `codetask_start_turn` again. Do not kill the planner OpenCode process; the execution tree is generated by that planner.
3. Poll only via `codetask_get_latest_job` / `codetask_get_job` / `codetask_get_plans` until a terminal planning outcome is visible (`plan_ready` / `ready_to_launch` / failed).
4. **On detected success or failure**, run a Node-side plan check through public APIs (do not invent tree JSON). If failed or check fails, call `codetask_continue_job` to let the **server** re-run OpenCode planner — **up to 3 retries**.
5. When check passes: inspect plan, confirm plan nodes if required, then `codetask_confirm_plan`.
6. Wait for the launched job and report. Checkpoint `plan_ready` / `plan_confirmed` as applicable.

## Allowed tools
Only Test MCP allowlist for the case.

## Forbidden behavior
- Do not invent plan JSON
- Do not write workspace files
- Do not resume create_task chat turns during plan generation
- Do not skip confirm-plan when the case requires it
- Do not kill planner OpenCode just because Node is polling

## Completion
Call `report_case_result` once with jobId/plan identifiers in artifacts.
